Recurrent Neural Network for Sentence Generation - Game of Thrones
Recently I created a RNN using Google's Python library TensorFlow and Keras. I designed this RNN to predict try to generate the next letter of a given character to generate text based off of a source. My most impressive results were from using all of the Game of Thrones novels together as a source. Using the RNN, I was able to generate a new "chapter" of Game of Thrones. While the new chapter doesn't make much sense contextually, the results are still impressive.
If you would like to take a look at what it generated, I made a link here to look at the results.


Lil' Lexie - HackDartmouth V Winner
Using natural language processing and some minor discrete mathematics, my friends and I created a rap battle bot that used Amazon's Alexa to both hear what the user is saying and to also rap back at the user. To generate sentences, we created a custom version of Markov Chains and "funneled" the results into a context-free grammar. The end result was a robot that could rap back at you using things that you said to it as a way to contextualize its rebuttal. Here is the DevPost blog that we posted this on.

Footnote - Chrome Extension, Front-End, Security, & Server Management
Footnote was the project that I did at my first hackathon. The idea behind Footnote was allowing users to comment on any web page, regardless of whether or not the page had a comments section on it. We accomplished this by generating a custom "page" per unique URL that all users could comment on and see comments on. We also thought it would be important to have accounts on the platform, so we created our own account system from scratch. Another obstacle hit us when we were creating an account system which was security. In order to create secure passwords for our accounts, we had to hash the password in such a way that even we couldn't read it. The end result was hack-y, but it did work. You can see more here.

Sam-Bot - Full-Stack, Database & API Integration
 You may have seen Sam-Bot on your way here - it's the chat bot I made for this website. I made Sam-Bot mostly for fun, but also as a way to answer questions that people might have about me that aren't listed anywhere else. Sam-Bot was a challenging project to implement. I used Google's Dialogflow to handle intents and user input, JavaScript and jQuery to make it look nice, and a little bit of php. I also had to rent out a remote Linux machine to run the code on, because all of the hosts that I tried to run it on only supported static programs, whereas Sam-Bot requires a dynamic site. I setup an Apache server on the Linux machine and then bought the domain samuel.tech to use as a custom URL. Overall, I am very pleased with the results.
